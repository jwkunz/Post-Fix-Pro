# Build Notes - v2.25.0

## Summary
- Bumped crate version from `2.24.0` to `2.25.0`.
- Replaced standalone Complex Entry widget with stack-driven complex conversion operators in the Complex panel:
  - `cart`: compose/decompose rectangular form.
  - `pol`: compose/decompose polar form (RAD/DEG aware).
  - `npol`: compose/decompose normalized polar form (`arg = 2πb`).
- Behavior for each button:
  - If top of stack is complex: decompose into two real scalars.
  - Else if top two stack values are real scalars: compose into one complex value.
- Removed obsolete complex-entry form controls and associated JavaScript glue.
- Updated help documentation for new complex conversion workflow.
- Added backend/API regression tests for `cart/pol/npol` compose+decompose behavior.
