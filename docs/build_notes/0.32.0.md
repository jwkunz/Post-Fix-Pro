# Build Notes - v0.32.0

## Summary
- Bumped crate minor version from `0.31.0` to `0.32.0`.
- Migrated backend matrix representation to complex values by default:
  - `Matrix.data` now stores complex entries.
  - Matrix algebra operations (`add`, `sub`, `mul`, scalar multiply, `det`, `inverse`, `solve`) now run in complex arithmetic.
- Updated determinant output for matrices to return a complex scalar.
- Updated API matrix serialization/input:
  - Matrix payload entries now use complex objects (`{ re, im }`).
- Updated frontend matrix entry path:
  - Matrix text input tokens are cast to complex on submit (`{ re: x, im: 0 }`).
  - Matrix rendering now formats complex entries.
- Updated help documentation to clarify that matrix entries are cast to complex values.
