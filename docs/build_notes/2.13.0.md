# Build Notes - v2.13.0

## Summary
- Bumped crate version from `2.12.0` to `2.13.0`.
- Fixed statistics behavior to match scalar-set intent:
  - `mean`, `mode`, `variance`, `std_dev`, `max`, `min` now work on either:
    - top vector matrix (`Nx1` or `1xN`), or
    - scalar-only stack (treated as one scalar set).
- Added helper logic to safely convert a scalar-only stack into a real vector view for stats ops.
- Preserved strict validation:
  - scalar-stack stats reject complex scalars with non-zero imaginary parts.
  - mixed stacks containing matrices and scalars still error unless top is a vector matrix.
- Updated help text to document stats support for scalar-only stack datasets.
- Added/updated tests for scalar-stack stats behavior in backend and API test suites.
